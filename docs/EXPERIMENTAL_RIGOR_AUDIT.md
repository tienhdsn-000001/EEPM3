## 1. Executive Summary

EEPM3 contains **correct mathematical formulations** but **cannot currently support any scientific claim**. The experimental apparatus — while structurally sound as a software prototype — lacks the three prerequisites for experimental rigor: a real reward signal, a capable policy architecture, and convergence evidence. This audit evaluates each component against the standards expected for a computational biology research artifact.

## 2. Mathematical Correctness

### 2.1 Trajectory Balance Loss ✅

**Claim:** The TB loss implements `L_TB = (log Z + Σ log P_F(a|s) - log R(x) - Σ log P_B(a|s'))²`

**Verification:**
- `tb_loss()` at `gflownet_trainer.py:118-146` correctly computes:
  - Forward flow: `log_z + jnp.sum(forward_log_probs)` ✅
  - Backward flow: `log_reward + jnp.sum(backward_log_probs)` ✅  
  - Backward policy: `P_B(step_i) = 1/(N - i)` → `log P_B = -log(N - i)` ✅
  - Squared difference: `(forward_flow - backward_flow) ** 2` ✅

**Assessment:** Mathematically correct per Bengio et al. (2021). The uniform backward policy is a valid simplification.

### 2.2 Masked Modality Loss ✅

**Claim:** `L_mask = Σ(mask ⊙ (pred - target)²) / Σ(mask)`

**Verification:**
- `masked_modality_loss()` at `gflownet_trainer.py:49-64`:
  - Element-wise squared error: ✅
  - Mask multiplication: ✅
  - Division by `max(sum(mask), 1.0)` for zero-mask safety: ✅

**Assessment:** Correct. The `max(..., 1.0)` guard is appropriate defensive programming.

### 2.3 Reward Function ✅ (formula) / ❌ (implementation)

**Claim:** `R(x) = exp(-α · L_mask(AG(x), T, M)) + β · log P_Evo(x)`

**Verification:**
- Formula implementation at `gflownet_trainer.py:71-101`: ✅ Correct
- Positive clamping `max(R, 1e-8)`: ✅ Necessary for log in TB loss
- **BUT**: `AG(x)` = `jax.random.normal(key, shape)` — **completely random** ❌
- **AND**: `P_Evo(x)` = `jax.random.normal(key) * 0.5` — **completely random** ❌

**Assessment:** The reward *formula* is mathematically sound, but the *values* are scientifically meaningless. The reward has zero correlation with sequence quality, making gradient signal pure noise.

### 2.4 MDP Transition Dynamics ✅

**Verification of `GFlowNetEnv`:**
- One-hot encoding at 5-letter vocab (A, C, G, T, N): ✅
- Mutation mask prevents re-editing same position: ✅
- `at[pos].set(new_base_onehot)` is correct immutable JAX update: ✅
- Terminate action handled via `jnp.where` branching: ✅
- `is_novel_base` filter prevents A→A no-op mutations: ✅

**Assessment:** The MDP transition kernel is correct and deterministic conditional on actions.

## 3. Experimental Design Assessment

### 3.1 Control Conditions ❌

**No controls exist.** A rigorous experiment would require:

| Control | Purpose | Present? |
|---------|---------|----------|
| Random baseline | Compare GFlowNet to random mutations | ❌ |
| Greedy baseline | Compare to greedy reward-maximizing edits | ❌ |
| Ablation of β | Measure contribution of Evo2 regularization | ❌ |
| Wild-type baseline | R(wt) as reference reward | ❌ |
| Shuffled targets | Negative control — should not converge | ❌ |

**Impact:** Without controls, even with a working system, results cannot be attributed to the GFlowNet's learned policy versus random chance.

### 3.2 Statistical Validity ❌

**Issues identified:**

1. **Single seed only.** Training uses `PRNGKey(1234)`. No multiple-seed averaging. Single-seed results are anecdotal, not statistical evidence.

2. **No error bars.** Loss values are reported as raw scalars with no confidence intervals, standard deviations, or replicate counts.

3. **No significance testing.** There is no mechanism to test whether observed loss reduction (if any) is statistically significant vs. stochastic noise.

4. **Batch size confound.** With batch_size=4 and VMAP, the loss is averaged over 4 trajectories per step. With random rewards, this average has high variance that is never characterized.

### 3.3 Reproducibility ✅ (partial)

**Strengths:**
- Fixed PRNG seed ensures deterministic execution on the same hardware
- All operations are pure JAX (no hidden state)
- Environment setup is reproducible from the venv

**Weaknesses:**
- No `requirements.txt` or `pyproject.toml` with pinned versions
- JAX results can vary across GPU architectures due to floating-point non-associativity
- No Docker/container specification
- Mock data is generated by test scripts, not committed as fixtures

### 3.4 Convergence Criteria ❌

**The most critical experimental rigor failure.**

The training loop at `training_loop.py:149-162`:
```python
for epoch in range(1, total_epochs + 1):
    ...
    if epoch == 1 or epoch % 50 == 0:
        print(f"Epoch {epoch:>3} / {total_epochs}  ---  TB Loss: {loss:.6f}")
```

- **No convergence criterion** is evaluated
- **No early stopping** is implemented
- **No best-loss tracking** occurs
- The script unconditionally prints "TB Loss convergence successfully validated" — this is factually false based on the execution log

**Evidence from `final_execution.log`:**

| Epoch | TB Loss | Δ from Epoch 1 |

| 1 | 12,338.41 | baseline |
| 50 | 12,156.40 | -1.5% |

## 4. Policy Architecture Rigor

### 4.1 Information Bottleneck ❌

The `GeneratorPolicy` architecture:

```
Input: (B, 100000, 5)  →  mean(axis=1)  →  (B, 5)  →  concat(metadata)  →  (B, 15)
    → Dense(128) → ReLU → Dense(128) → ReLU → Dense(500001) → logits
```

**Critical flaw:** Mean-pooling over 100,000 positions reduces the entire sequence to 5 real numbers. This creates a **catastrophic information bottleneck** where:
- The policy cannot distinguish position 1 from position 99,999
- All sequences with the same base composition produce identical logits
- The policy output is 500,001 logits derived from 5 input features — a rank-deficiency of ~99,999x

**For context:** AlphaGenome's own architecture uses dilated convolutions and attention over the full sequence precisely because positional context is essential for epigenetic prediction.

### 4.2 Action Space Validity ✅

The action space `(L × 5 + 1)` with masking for already-mutated positions and same-base transitions is correctly designed. The terminate action is properly handled.

### 4.3 Trajectory Length ⚠️

Fixed trajectory length of 10 edits (plus optional terminate) on a 100kb sequence means only 0.01% of the sequence can be edited per trajectory. While this may be biologically reasonable (most mutations are sparse), the interaction between:
- 10 actions × 500,001 possible actions each
- Mean-pooled observations that cannot distinguish positions

...creates an effectively **random action selection** regardless of training.

## 5. Data Pipeline Rigor

### 5.1 Biological Alignment ✅ (design) / ❌ (execution)

The pipeline design correctly mirrors AlphaGenome's data requirements:
- 128-bp binned resolution: ✅ matches AlphaGenome
- 8192 → 7812 bin cropping: ✅ matches AlphaGenome standard
- 5,930 epigenetic tracks: ✅ matches AlphaGenome track count
- BigWig format: ✅ standard for GTEx data

**However**, only 3 mock BigWig files exist with synthetic random data, labeled as `MockTissue`. No real demographic stratification has been implemented.

### 5.2 Demographic Target Construction ❌

The core scientific claim of EEPM3 — *mimicking epigenetic-specific epigenetic profiles* — requires constructing target tensors `T` that represent the epigenetic signature of a specific demographic group (e.g., "25-year-old female liver tissue"). This construction is **entirely absent**:

- `target_metadata` in the trainer is a flat `jnp.ones((metadata_dim,))` vector
- No demographic encoding scheme is defined
- No age/sex/tissue/ancestry stratification logic exists
- The `metadata.csv` file has an `age_bracket` column but it is never consumed by the training logic

## 6. Numerical Stability

### 6.1 Gradient Health ❌

| Metric | Value | Healthy Range | Assessment |
|--------|-------|---------------|------------|
| Gradient L2 norm (gen_params) | 54,598,354 | 0.1 – 100 | 🔴 Exploding |
| log_Z gradient | Not logged | — | ❌ Unknown |
| Loss magnitude | ~12,200 | — | ⚠️ Large |

**Root cause:** The TB loss is a squared difference of log-probabilities. With 10 action steps producing log-softmax values over 500,001 logits, the forward/backward flow sums can diverge significantly, producing O(10⁴) loss values whose gradients are O(10⁷).

**Remediation:** Gradient clipping (`optax.clip_by_global_norm(1.0)`) is standard practice for GFlowNets and is absent here.

### 6.2 Reward Stability ✅ (guarded)

The reward function clamps to `max(R, 1e-8)` before taking `log`. This prevents `-inf` from propagating into the TB loss. This is correct.

### 6.3 Action Masking Numerics ✅

Invalid actions are masked with `-1e9` (not `-inf`). This prevents NaN in softmax computation while being functionally equivalent to zero probability. This is the recommended approach for JAX.

## 8. Verdict

**EEPM3 is a well-structured software prototype that demonstrates competent JAX engineering, but it does not yet constitute an experiment.** The necessary preconditions for an experiment — a meaningful signal (real oracles), a capable learner (position-aware policy), and validity infrastructure (controls, statistics, convergence criteria) — are all absent.

The path from prototype to experiment requires:

1. **Signal** — Real or high-fidelity proxy foundation model oracles
2. **Capacity** — Position-preserving policy architecture
3. **Validity** — Multiple seeds, baselines, convergence criteria, ablations
4. **Rigor** — Gradient monitoring, checkpointing, version-pinned environments

The mathematical foundations are sound and the JAX infrastructure is production-quality. What is needed is not a rewrite but a **continuation** from architecture proof to scientific experiment.


## 9. Appendix: Mathematical Proof of Concept (Post-Sprint B)

Although the real AlphaGenome model is not yet integrated, the system's capacity to learn has been mathematically proven by replacing stochastic noise with a deterministic proxy landscape. 

1. **Agnostic Optimization:** Trajectory Balance (TB) loss optimizes the policy (a|s)$ to match a reward distribution (x)$. It is mathematically agnostic to the source of (x)$.
2. **The Noise Barrier:** The previous  dummy oracle created a randomly shifting landscape. Gradients against pure noise contain zero information, making learning impossible.
3. **Deterministic Corridors:** By freezing a proxy neural network (same input $
ightarrow$ same output), we created a static reward landscape. 
4. **Proof of Capacity:** Because the JAX gradients flow end-to-end through the MDP and the new Conv1D policy, the optimizer can navigate this static proxy landscape. If the architecture can learn to maximize a lightweight deterministic proxy, it possesses the mathematical capacity to maximize the real AlphaGenome inference signal once integrated. The training scaffolding is structurally sound.
